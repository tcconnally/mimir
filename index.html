What Mimir Does

AI agents forget everything between sessions. Mimir gives them durable memory — structured entities that survive restarts, journal events that log decisions, and state that expires when it should. All through standard MCP tools. No API keys. No cloud. A single static binary.

Features

Structured Entity Model

Idempotent storage by (category, key). Update the same entity as many times as you want — no duplicates.

Ebbinghaus Decay

Memories fade when unused, strengthen on recall. Layer progression: buffer → working → core.

Journal Events

Append-only log with evaluated/acted/forward structure. Timeline queries for decision auditing.

State with TTL

Key-value store with automatic expiration. Session state, counters, locks — gone when they should be.

.md Vault

Export entities as markdown files with YAML frontmatter. Human-readable, git-trackable, Obsidian-compatible.

Entity Linking

Create navigable relationship graphs between entities. Chain traversal follows links up to configurable depth.

Near-Duplicate Detection

Trigram overlap check prevents storing the same fact twice. Bumps existing entity instead of creating clutter.

Conflict Detection

Finds contradictory entities in the same category. Flags low-similarity pairs for agent review.

Schema
